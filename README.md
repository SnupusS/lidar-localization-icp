# lidar-localization-icp

**Локализация робота по заранее построенной карте методом ICP: итеративное совмещение скана лидара с картой через KD-дерево и SVD. Два режима — реальное время и постфактум.**

## Описание

Алгоритм определяет положение робота `(x, y, θ)` на известной карте, итеративно совмещая текущий скан лидара со стенами карты методом ICP (Iterative Closest Point). Поиск ближайших точек ускорен KD-деревом, оптимальное преобразование на каждой итерации вычисляется через SVD. Работает в двух режимах: с реальным лидаром Hokuyo по Ethernet и постфактум по записанному лог-файлу сырых данных — во втором случае восстанавливается полная траектория движения.

## Алгоритм

1. **Загрузка карты.** Карта `lines.bin` — полилинии (стены) из CloudCompare. Они растеризуются в облако опорных точек, по которому строится KD-дерево для быстрого поиска ближайших соседей.
2. **Инициализация по клику.** Пользователь указывает на карте примерную точку, где находится лидар. Курс алгоритм находит сам — грубым перебором по XY-сетке и углу с оценкой качества по KD-дереву, затем уточняет ICP. Зачем: ICP сходится только из близкого начального приближения, и подсказка пользователя задаёт это приближение.
3. **ICP-совмещение.** Для каждой точки скана ищется ближайшая точка карты (KD-дерево); по парам соответствий через SVD вычисляется жёсткое преобразование (поворот + сдвиг), минимизирующее суммарное расстояние; преобразование применяется к позе; шаги повторяются до сходимости. Используется усечённый (trimmed) ICP — часть наихудших соответствий отбрасывается, что делает оценку устойчивой к динамическим помехам и выбросам.
4. **Трекинг с защитой от телепортации.** В установившемся режиме поза прогнозируется моделью постоянной скорости, затем уточняется ICP. «Motion gate» не позволяет позе/курсу прыгнуть дальше физически возможного за один кадр — отсекаются ложные совмещения. При потере (рост ошибки) алгоритм автоматически переходит к повторной грубой инициализации.

## Технический стек

|Категория|Компоненты|
|-|-|
|Лидар|Hokuyo UTM-30LX-EW, SCIP 2.2 (команда MD), 1080 точек/скан, FOV 270°|
|Связь|Ethernet / TCP (live-режим)|
|Карта|CloudCompare `.bin` (полилинии стен), масштаб 0.05 м/у.е.|
|Алгоритмы|ICP (KD-дерево + SVD), грубый перебор инициализации, motion gate|
|Библиотеки|NumPy, SciPy (`cKDTree`), OpenCV (визуализация)|

## Структура файлов

|Файл|Описание|
|-|-|
|`localize.py`|ICP-локализация целиком: декодер SCIP, загрузка карты, грубая инициализация, ICP (KD-дерево + SVD), трекинг с motion gate, два источника данных (live-лидар / файл), OpenCV-визуализация|
|`lines.bin`|Бинарная карта местности (полилинии стен из CloudCompare)|
|`podval.txt`|Пример сырых данных реального лидара Hokuyo (формат SCIP 2.2, 3-символьное ASCII-кодирование расстояний)|
|`requirements.txt`|Зависимости Python|

## Как запустить

1. **Установить зависимости:**

```bash
   pip install -r requirements.txt
   ```

2. **Режим постфактум (по записи, без лидара):**

```bash
   python localize.py --file podval.txt
   ```

или с указанием частоты и карты:

```bash
   python localize.py --file podval.txt --hz 25 --map lines.bin
   ```

3. **Режим реального времени (с лидаром Hokuyo по Ethernet):**

```bash
   python localize.py --live
   ```

4. **Интерактивный выбор режима** (если запустить без флагов):

```bash
   python localize.py
   ```

5. **Управление в окне:**

   * **клик** — задать стартовую точку (где примерно находится лидар);
   * **Q / ESC** — выход;
   * **R** — принудительное восстановление (повторная инициализация);
   * **ПРОБЕЛ** — пауза/продолжение (в режиме file).

   ## Параметры

   Настройки — в начале файла `localize.py` (блок «НАСТРОЙКИ»).

|Параметр|Значение|Назначение|
|-|-|-|
|`MAP\\\_FILE`|`lines.bin`|Карта по умолчанию|
|`STEPS`|1080|Точек в скане|
|`ANGLE\\\_MIN/MAX\\\_DEG`|−135 … +135|Угловой диапазон (FOV 270°)|
|`--hz`|25|Частота воспроизведения в режиме file|
|`MIN/MAX\\\_RANGE\\\_M`|0.20 … 30.0|Валидный диапазон дальности, м|

Точные численные значения параметров ICP (число итераций, trim-доля) и грубого перебора заданы константами в начале `localize.py` — при необходимости подстройте под свою карту и динамику робота.

## Известные ограничения

* ICP сходится только из близкого начального приближения — нужна разумная стартовая подсказка кликом; в больших симметричных помещениях возможна неоднозначность.
* Стеклянные и зеркальные поверхности дают ложные отражения и ухудшают совмещение.
* Карта считается статической: крупные изменения обстановки снижают точность.
* В режиме file качество зависит от полноты и корректности записанного лог-файла.
* При длительной полной окклюзии или сильном дефиците геометрических признаков (длинный пустой коридор) возможна кратковременная потеря с переходом к повторной инициализации.
